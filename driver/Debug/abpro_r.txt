                       О программе
1 Общие положения и функциональное назначение
Программа driver.exe была создана с целью демонстрации знаний, 
приобретенных обучаемым в области технологии разработки 
объектно-ориентированных программ средствами языка С++. 
Программа была построена на основе объектов предметной области 
"Водитель". Ее работа заключается в демонстрации реализации принципов
ООП и управлении поведением объектов.Данная программа может 
эффективно функционировать на любом персональном компьютере с 
процессором семейства Intel, или совместимым с ним по набору 
инструкций процессором.В обязательные требования также входит
операционная система Windows 9x, XP,7, 200x,  32 Мб ОЗУ (желательно
64 Мб) и 10 Мб свободного места (программа + временные файлы
программы в сумме занимают 1,5 Mб в файловой системе NTFS). Для 
работы с данным проектом требуется специалист в области логистики,
либо рядовой пользователь, желающий попробовать себя в роли
администратора транспортной системы с помощью созданной программы.  
2 Описание логической структуры
Всего в системе существует семь классов (Person, Driver, LicenseTag, B, B1, BE, 
Auto), для четырех из которых возможно формировать объекты (класс 
LicenseTag, B1, BE не могут порождать объекты). Пользователь имеет 
возможность создавать, изменять (в том числе с помощью перегруженных 
операторов) экземпляры классов, сохранять состояние системы в файл, 
просматривать его. Класс Driver наследует свои поля от класса Person. Класс 
Auto является дружественным классу Driver. Класс LicenseTag – абстрактный
класс, методы которого наследуются классам B1, BE , а их методы наследуются
в свою очередь классу B.
3 Используемые технические средства
При создании программы была использована среда разработки Visual Studio
2008. Visual Studio предоставляет полный набор инструментов и служб для 
создания различных приложений, как для платформы Microsoft, так и для
других платформ с использованием языка С++. Программа тестировалась на
ОС Windows 7.  Клавиатура, манипулятор типа «мышь»,  устройство типа 
«стандартный монитор» с разрешением 800x600 пикс.
4 Вызов и загрузка
Запуск программы выполняется с помощью вызова исполнимого файла 
driver.exe (в среде MS-DOS или Windows 9x/200x). После запуска программы
на экране появляется окно заставки, которое содержит общую информацию
о теме курсовой работы и авторе. Для успешной работы приложения 
необходимо наличие всех текстовых файлов с записями
интерфейса. Необходимым является и файл конфигурации config.txt, в 
котором хранится путь доступа к файлу результатов.
5 Входные и выходные данные
Все вводимые значения проверяются на корректность. При ошибке ввода 
пользователю предоставляется возможность повторить попытку ввода, 
которая позволяет продолжить работу. Все записи интерфейса читаются 
из файлов. Программа имеет два языка: русский и английский.
Смена языка возможна в любом месте программы.
Пользователь в любой момент на экран может вывести содержимое файла, 
в котором будет храниться состояние системы.
